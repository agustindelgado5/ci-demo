# Introducción

La Integración Continua (CI) es una práctica fundamental dentro de la ingeniería de software moderna. Forma parte del enfoque DevOps y consiste en integrar cambios de código de manera frecuente en un repositorio compartido, ejecutando procesos automáticos como compilación y pruebas.

El objetivo principal de la CI es detectar errores de manera temprana, mejorar la calidad del software y reducir los problemas de integración entre diferentes partes del sistema.

En este trabajo se presenta una implementación práctica de Integración Continua utilizando GitHub Actions aplicada a un proyecto simple desarrollado en C#.
